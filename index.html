TypeScript

Into to TypeScript

TypeScript is a strongly types scripting language. It adds additional features and syntax to JavaScript. TypeScript adds static typing to JavaScript which means that developers can add types when using TypeScript. TypeScript is a handy landuage to use when dealing with JavaScript and can help catch errors much earlier then JavaScript can.

TypeScript vs JavaScript Example

Like mentioned above TypeScript is JavaScript with added functions. Below are some code examples that show the difference between the two languages

Hello World in TypeScript

let message: string = 'Hello, World!';
console.log(message);

Hello World in JavaScript

console.log('Hello World');

Defining Types

TypeScript supports an extension of the JavaScript language, which offers places for you to tell TypeScript what the types should be.

For example, to create an object with an inferred type which includes name: string and id: number, you can write:

const person = {
    name: 'Emily",
    birth-month: 3,
    birth-day: 12,
};

You can explicitly describe this object’s shape using an interface declaration:

interface Person {
    name: string;
    birth-month: number,
    birth-day: number,
}

You can then declare that a JavaScript object conforms to the shape of your new interface by using syntax like : TypeName after a variable declaration:

const user: Person = {
    name: "Emily",
    birth-month: 3,
    birth-day: 12,
};

If you provide an object that doesn’t match the interface you have provided, TypeScript will warn you 

interface Person {
    name: string;
    birth-month: number;
    birth-day: number;
}
    
const user: Person = {
    username: "Emily",
    Type '{ username: string; birth-month: number; birth-day: number; }' is not assignable to type 'Person'.
    Object literal may only specify known properties, and 'username' does not exist in type 'Person'.
    birth-month: 3,
    birth-day: 12,
};

Generics

Generics provide variables to types. A common example is an array. An array without generics could contain anything. An array with generics can describe the values that the array contains..

type StringArray = Array;
    type NumberArray = Array;
    type ObjectWithNameArray = Array<{ name: string }>;

You can declare your own types that use generics:

interface Backpack {
    add: (obj: Type) => void;
    get: () => Type;
}
    
    // This line is a shortcut to tell TypeScript there is a
    // constant called `backpack`, and to not worry about where it came from.
    declare const backpack: Backpack;
    
    // object is a string, because we declared it above as the variable part of Backpack.
    const object = backpack.get();
    
    // Since the backpack variable is a string, you can't pass a number to the add function.
    backpack.add(23);
    Argument of type 'number' is not assignable to parameter of type 'string'.

Classes

TypeScript offers full support for the class keyword introduced in ES2015. As with other JavaScript language features, TypeScript adds type annotations and other syntax to allow you to express relationships between classes and other types.

Here’s the most basic class - an empty one:

class Count {}

A field declaration creates a public writeable property on a class:

class Count {
    x: number;
    y: number;
    z: number;
}
    
const num = new Count();
num.x = 6;
num.y = 2;
num.z = num.x + num.y;

Compiling and Running

There are a couple steps in running and compiling TypeScript code.

The frist is to install TypeScript. This can be done with the following command

npm install -g typescript

You can test your install by checking the version or help.

tsc --version
tsc --help

Next we can build our Typescript file

tsc FileName.ts

Or in some cases the following will work to build and compile your code

npm run compile
